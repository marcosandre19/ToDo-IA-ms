# Decisões técnicas (seguir exatamente)

## Biblioteca
- **`springdoc-openapi-ui` versão 1.7.0** (compatível com Spring Boot 2.7.x; versões 2.x são pra Spring Boot 3+)
- Adicionar dependência ao `pom.xml`:
```xml
  <dependency>
      <groupId>org.springdoc</groupId>
      <artifactId>springdoc-openapi-ui</artifactId>
      <version>1.7.0</version>
  </dependency>
```
- **NÃO usar** `springfox-swagger2` (descontinuado), `springdoc-openapi-starter-webmvc-ui` (versão 2.x — incompatível com Spring Boot 2.7).

## Configuração no `application.properties`
Adicionar:
```properties
# OpenAPI / Swagger
springdoc.api-docs.path=/v3/api-docs
springdoc.swagger-ui.path=/swagger-ui.html
springdoc.swagger-ui.operations-sorter=method
springdoc.swagger-ui.tags-sorter=alpha
springdoc.show-actuator=false
```

- `operations-sorter=method` agrupa as operações por verbo HTTP, mais previsível que ordem alfabética
- `show-actuator=false` mantém `/actuator/health` fora da documentação da API (é infra, não domínio)

## Anotações no `TarefaController`

### Tag da classe
- `@Tag(name = "Tarefas", description = "Operações de gerenciamento de tarefas")` na classe
- Sem `@Tag` a UI exibe nome derivado do nome da classe ("tarefa-controller"), que fica feio

### Por endpoint
Cada método do controller deve ter:
- `@Operation(summary = "...", description = "...")` — `summary` é a linha curta na UI; `description` opcional, só se agregar
- `@ApiResponses(value = { @ApiResponse(...), @ApiResponse(...) })` — listar todos os status codes que o endpoint pode retornar, com `responseCode` e `description`

Status codes a documentar **conforme `escopo-todo.md`** (e código real do controller):

| Endpoint | Status documentados |
|----------|---------------------|
| `POST /tarefas` | 201, 400 |
| `GET /tarefas` | 200 |
| `GET /tarefas/{id}` | 200, 404 |
| `PUT /tarefas/{id}` | 200, 400, 404 |
| `DELETE /tarefas/{id}` | 204, 404 |
| `PATCH /tarefas/{id}/status` (ou path real) | 200, 400, 404 |

Para endpoints que retornam erro estruturado (4xx), incluir `content = @Content(schema = @Schema(implementation = ErroResponse.class))` no `@ApiResponse` correspondente — assim a UI mostra o formato do JSON de erro.

### Textos das anotações
- **Em pt-br**, consistente com o resto do projeto
- Tom técnico, direto, sem floreio
- `summary`: imperativo curto ("Cria uma nova tarefa", "Lista todas as tarefas", "Busca tarefa por identificador")
- `description` opcional — usar apenas se houver detalhe relevante além do `summary`. Não duplicar.

## Anotações nos DTOs (`TarefaRequest`, `TarefaResponse`, `AtualizarStatusRequest`, `ErroResponse`)

Em cada campo dos DTOs, adicionar:
- `@Schema(description = "...", example = "...")`
- `description` em pt-br
- `example` com valor realista (não `"string"`, não `"example"`) — exemplo: `example = "Comprar leite"`, `example = "PENDENTE"`, `example = 1`
- Para enums: a UI já lista os valores possíveis automaticamente; descrever só o significado

Exemplo:
```java
@Schema(description = "Título da tarefa", example = "Comprar leite", maxLength = 100)
@NotBlank(message = "O título é obrigatório")
@Size(max = 100, message = "O título deve ter no máximo 100 caracteres")
private String titulo;
```

**Não duplicar** restrições já expressas em Bean Validation (`@NotBlank`, `@Size`) — o springdoc lê essas anotações automaticamente. `@Schema` adiciona apenas descrição e exemplo.

### Schema da entidade `Tarefa`
A entidade JPA **não** deve receber `@Schema` — ela é interna, não faz parte do contrato da API. Documentação fica nos DTOs.

## Bean de configuração (opcional, mas recomendado)
Criar `OpenApiConfig.java` em `config/` (criar pacote se não existir) com metadados gerais da API:

```java
@Configuration
public class OpenApiConfig {

    @Bean
    public OpenAPI tarefasOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("API de Gerenciamento de Tarefas")
                .description("MVP de API REST para gerenciar tarefas (CRUD).")
                .version("1.0.0"));
    }
}
```

Isso preenche o cabeçalho da UI com nome e descrição do projeto. Sem essa config, aparece "OpenAPI definition" genérico.
