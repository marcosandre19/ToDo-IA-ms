# Restrição arquitetural mandatória — isolamento de dependências

**Os testes do controller NÃO devem tocar em repositório nem em banco de dados.** Estado global entre testes é causa principal de suítes flakey, e essa etapa precisa ficar imune a isso por construção, não por convenção.

A forma de garantir isso é usar **`@WebMvcTest(TarefaController.class)`** com **`@MockBean private TarefaService tarefaService`**. Essa combinação:
- Sobe APENAS a camada web (controller, handler global, conversão JSON, Bean Validation, filtros relacionados)
- Substitui o `TarefaService` no contexto por um mock — o service real **nunca é instanciado**
- Como o service é mockado, repositório e banco **não fazem parte do contexto do teste**, então não há estado para vazar entre testes

**NÃO usar `@SpringBootTest`** — sobe o contexto inteiro, expõe o teste a estado global do banco e quintuplica o tempo de execução.

**NÃO usar `@DataJpaTest` aqui** — é para testes de repository, não de controller.

# Decisões técnicas (seguir exatamente)

## Anotações da classe de teste
```java
@WebMvcTest(TarefaController.class)
class TarefaControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private ObjectMapper objectMapper;

    @MockBean
    private TarefaService tarefaService;

    // testes
}
```

- `@WebMvcTest(TarefaController.class)` — limita o slice ao controller específico, não escaneia o resto
- `@MockBean` — injeta o mock como bean Spring, substituindo o real (DIFERENTE de `@Mock` puro do Mockito)
- `MockMvc` e `ObjectMapper` vêm autoconfigurados pelo `@WebMvcTest`
- O `GlobalExceptionHandler` é detectado automaticamente porque está anotado com `@RestControllerAdvice` — não precisa importá-lo manualmente

## Convenção de nomes
- Padrão: `deve<Comportamento>Quando<Contexto>` em pt-br, sem acentos no nome do método
- Exemplos:
  - `deveCriarTarefaERetornar201ComLocation`
  - `deveRetornar400QuandoCriarComTituloVazio`
  - `deveRetornar404QuandoBuscarTarefaInexistente`
  - `deveRetornar400QuandoBodyForJsonInvalido`
- `@DisplayName` em pt-br **com** acentos é bem-vindo em testes-chave para legibilidade no relatório

## Padrão AAA dentro de cada teste
Estruturar cada teste em três blocos separados (Arrange, Act, Assert) com comentários ou linha em branco. `MockMvc.perform(...)` encadeia Act e Assert na mesma chamada, mas o Arrange (configuração de mocks e payloads) deve ficar visualmente separado.

## Cobertura mínima exigida

### `POST /tarefas`
- **Sucesso (201)**: payload válido, `tarefaService.criar(...)` mockado retorna `Tarefa` com ID populado. Asserções:
  - `status().isCreated()` (201)
  - `header().string("Location", containsString("/tarefas/"))` — header presente e contendo o path
  - `jsonPath("$.id").value(<id esperado>)`
  - `jsonPath("$.titulo").value(<titulo esperado>)`
  - `verify(tarefaService).criar(any(Tarefa.class))`
- **Validação falhou (400)**: payload com titulo vazio (ou outra regra do escopo). Asserções:
  - `status().isBadRequest()`
  - `jsonPath("$.status").value(400)`
  - `jsonPath("$.camposComErro.titulo").exists()` — confirma que erro por campo foi serializado (ajustar nome do campo conforme `ErroResponse` real)
  - **`tarefaService` NUNCA foi chamado**: `verify(tarefaService, never()).criar(any())`
- **JSON malformado (400)**: body com JSON sintaticamente inválido (ex.: `"{ titulo: }"`). Asserções:
  - `status().isBadRequest()` — confirma que `HttpMessageNotReadableException` é tratado pelo handler global
  - `verify(tarefaService, never()).criar(any())`
  - **Se este teste falhar com 500**, sinalize no chat: o `GlobalExceptionHandler` está incompleto e precisa de `@ExceptionHandler(HttpMessageNotReadableException.class)`. **Pare e me avise — não corrija o código de produção neste prompt**

### `GET /tarefas`
- **Lista com tarefas (200)**: `tarefaService.listar()` retorna `List.of(tarefa1, tarefa2)`. Asserções:
  - `status().isOk()`
  - `jsonPath("$").isArray()`
  - `jsonPath("$.length()").value(2)`
  - `jsonPath("$[0].id").value(...)`
- **Lista vazia (200)**: `tarefaService.listar()` retorna `Collections.emptyList()`. Asserções:
  - `status().isOk()`
  - `jsonPath("$.length()").value(0)`

### `GET /tarefas/{id}`
- **Sucesso (200)**: `tarefaService.buscarPorId(1L)` retorna a tarefa. Assertions sobre o JSON.
- **Não encontrado (404)**: `tarefaService.buscarPorId(99L)` lança `TarefaNaoEncontradaException`. Asserções:
  - `status().isNotFound()`
  - `jsonPath("$.status").value(404)`
  - `jsonPath("$.mensagem").value(containsString("99"))` — mensagem da exceção mencionando o ID
  - `jsonPath("$.caminho").value("/tarefas/99")`

### `PUT /tarefas/{id}`
- **Sucesso (200)**: payload válido, `tarefaService.atualizar(...)` retorna a tarefa atualizada. Assertions sobre o JSON retornado.
- **Não encontrado (404)**: service lança `TarefaNaoEncontradaException`. Mesmas asserções do GET 404.
- **Validação falhou (400)**: payload inválido. Mesmas asserções do POST 400.

### `DELETE /tarefas/{id}`
- **Sucesso (204)**: `tarefaService.apagar(1L)` retorna `void`. Asserções:
  - `status().isNoContent()` (204)
  - `verify(tarefaService).apagar(1L)`
- **Não encontrado (404)**: `tarefaService.apagar(99L)` lança `TarefaNaoEncontradaException`. Asserções de 404.
